个人博客（一）—— Hexo+Github Pages搭建个人博客

发表于 2021-10-06

借助Hexo官方文档的介绍，Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。这篇文章介绍如何快速部署一个可以访问的博客网站。

教程基本基于官方文档，我的部署基于Windows环境。需要搭建的第一步是环境配置，主要配置Node.js和Git环境。

Git安装

Windows下直接访问Git下载页面下载最新版本并安装。

下载完成之后需要进行一些初始化配置（在Git bash中）：

  • 首先设置姓名和邮箱地址，名字请用英文输入：

    git config --global user.name "Firstname Lastname"

    git config --global user.email "your_email@example.com

  • 将color.ui设置为auto以提高命令输出的可读性：

    git config --global color.ui auto

  • 中文乱码问题，设置这条命令之后就可以在git命令提示符中正常使用中文：

    git config --global core.quotepath false

Node.js安装

Windows下直接访问官方下载页面进行下载最新版本，选择下载64位msi格式的安装包，在使用Node.js官方安装程序时，请确保勾选Add to PATH选项（默认已勾选）。

安装Hexo

上面的应用程序安装完成后，即可使用npm安装Hexo。

npm install -g hexo-cli

建站

安装Hexo完成后，执行以下命令，Hexo将会在指定文件夹中新建所需要的文件。

此时Hexo基本的博客框架已经搭建完成，接下来进行Github Pages搭建。

创建仓库

创建一个名称为username.github.io的仓库，其中username是你的用户名，之后你的网站将会通过username.github.io域名进行访问。

添加ssh key

Github上连接已有仓库时的认证，是通过使用了SSH的公开密钥认证方式进行的，必须是项目的拥有者才可以添加SSH Key并使用ssh方式克隆仓库。现在让我们创建公开密钥认证所需的SSH Key，并将其添加至Github。

  • 运行下面的命令创建SSH Key：

    ssh-keygen -t rsa -C "your_email@example.com"

    "your_email@example.com"的部分请改成在创建账户时用的邮箱地址。在认证时需要设置密码，可以设置简单一些的密码。

  • 添加公开密钥：

    点击Github页面右上角的账户设定按钮（Account Setting），选择SSH Keys菜单，点击Add SSH Key，会出现Title和Key两个输入框。在Title中输入适当的密钥名称。Key部分粘贴你的公开密钥，公开密钥可以通过输入命令来获取：

    cat ~/.ssh/id_rsa.pub

    将返回的内容粘贴回去就可以添加成功，完成以上设置之后就可以用手中的私人密钥与Github进行认证和通信了。

  • SSH连接测试：

    ssh -T git@github.com

一键部署

Hexo提供了快速方便的一键部署功能，让您只需一条命令就能将网站部署到服务器上。

hexo deploy or hexo d

  1. 首先安装hexo-deployer-git：

npm install hexo-deployer-git --save

  1. 修改配置：

在 _config.yml 文件的最后，修改deploy参数。

完成之后就可以一键部署博客到Github Pages上，并使用 username.github.io 访问自己的博客。

Hexo常用命令

下面仅仅列出少数几个我最常用的命令，详细信息可以查看官方文档

  • hexo clean：清除缓存文件和之前构建的静态文件，在修改或编写新的文章后需要使用该命令进行更新。
  • hexo generate：生成静态文件，网站会根据这些静态文件进行显示。可以简写为hexo g。
  • hexo server：在本地启动服务器。默认情况下，访问网址为：http://localhost:4000/，可以通过该命令来查看自己网站的构建情况，在满意之后再进行Github Pages部署。可以简写为hexo s。
  • hexo d ：之前介绍过，部署网站。
  • hexo new "My new Post"：创建新的博客文章，创建之后可以在文件夹source/_post中找到该文章，之后使用Markdown文本编辑器（推荐Typora或VS code）进行编写。
